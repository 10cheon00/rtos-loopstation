# 테스트 과정

빠르게 테스트하기 위해 모든 코드를 codex에게 작성하도록 시켰고, 테스트 이후에는 과감히 지운 다음 설계 문서를 따라 구현할 계획이다.

## LCD 모듈 테스트

LCD가 배달 완료 되어서, 간단한 납땜을 마치고 브레드보드에 장착 후 테스트를 진행했다. 직접 LCD 출력 라이브러리를 구현하기에는 시간낭비같아 오픈소스인 *u8g2*를 사용하기로 했다. LCD가 128x64 해상도이고 단순 픽셀 출력만 가능한 모듈이어서 *lvgl*은 적절하지 않다고 판단했다. 

*u8g2*라이브러리에는 다양한 LCD 모듈을 다루는 코드가 있어서, ST7565 모듈만 다루는 코드를 제외하고 나머지 모듈들에 대한 파일들은 전부 제거했다. 그 다음 라이브러리의 .c파일과 .h파일들을 모두 최상위 CMakeListst.txt에 추가했다. .c파일들은 직접 하나씩 다 추가하고, .h파일들은 include에 경로만 등록했다. 

초기화 코드는 [이 영상](https://youtu.be/wXDFW6NsDK4)을 참고했다. 왜 erc12864~ 함수를 써야하는지는 부품과 관련된 내용이어서 잘 모르겠다. 

*u8g2*라이브러리는 여러 LCD 모듈들을 동일한 함수로 제어가능하게 해준다. 단, FatFs를 사용할 때 직접 SD카드에 명령을 보내고 받는 함수는 직접 작성해야했듯, 이 라이브러리를 쓸 때도 모듈에 데이터를 보내는 방법은 직접 구현해야한다. 그러나 메세지 프레임을 구성하는 건 라이브러리에서 담당하기 때문에, 실제 전송을 처리하는 계층만 사용자가 직접 구현하면 된다. codex가 작성해준 코드를 테스트함으로 간단하게 테스트에 성공했다.

## 버튼, 노브, 로터리 엔코더 테스트

LCD에 글자를 출력할 수 있게 되면서 다른 모듈들도 테스트하기 편해졌다. 입력된 데이터를 LCD에 출력함으로 결과를 바로 확인할 수 있게 됐다. 먼저 버튼들부터 입력이 잘 되는지, IFX/TFX에 연결된 LED들도 출력이 잘 되는지 테스트했다. 대부분의 GPIO들이 MCP23017에 물려져있으므로 I2C를 사용하고, MCP23017의 주소값에 따라 메세지 프레임을 구성한 뒤 입력된 버튼을 LCD에 출력함으로 테스트에 성공했다. 

노브와 슬라이더는 바로 보드의 ADC에 연결했고 이것 역시 LCD에 출력함으로 테스트에 성공했다.

로터리 엔코더도 바로 보드의 타이머에 연결했고 타이머를 엔코더 모드로 동작하게 만들어 테스트에 성공했다. 로터리 엔코더는 핀헤더가 윗 방향으로 나있고 90도 꺾여 있어서, 브레드보드에 장착하면 돌리기 어려운 구조다. 이 문제는 나중에 개선해야한다.

## SAI(INMP441 -> PCM5102A) 테스트

아직 SAI가 어떤 구조로 동작하는지 잘 모르겠지만, codex가 하라는 대로 핀 설정을 마치고 Block A에 PCM5102A를, Block B에 INMP441을 연결한 후 입력된 신호를 그대로 내보내도록 구현해보았다. 테스트에는 성공했지만, PCM5102A가 올바르게 작동하지 않아서 급하게 UDA1334A로 교체했다. 가지고 있는 PCM5102A가 브레드보드에 장착하기 어려운 구조라서 새로운 모듈을 주문했고, 배달되기 전까지는 UDA1334A로 테스트할 계획이다.

> SAI에 대해서 더 공부해야겠다.
